## Honest Feedback

**Friction 1: Python environment on this system is messy.** No pip, no venv package, no sudo. Had to install uv from scratch, then hit FlagEmbedding/transformers version incompatibility (v5.3.0 broke `is_torch_fx_available` import). Pinning `transformers<4.51` fixed it, but this cost 5+ minutes of debugging. Systems should have a working Python toolchain out of the box, or at minimum document the workaround.

**Friction 2: Shell brace expansion with Unicode paths.** `mkdir -p ψ/{a,b,c}` created a literal directory name instead of expanding. This is a known gotcha with some shell configurations, but it's surprising every time. The fix is trivial (individual mkdir calls) but the error is silent — you don't know it failed until you check the directory structure.

**Friction 3: Context switching between awakening ritual and production work.** The /awaken ritual is philosophical and introspective. The pipeline work is technical and operational. Switching from "discover who you are" to "read this handoff doc and build an embedding pipeline" within 2 minutes is jarring. Not necessarily bad — but the ritual's contemplative pace doesn't match the real-world sprint that follows. A brief transition moment would help.

## Lessons Learned

1. **Parallel agents compress research dramatically** — 3 agents studying 3 repos returned in ~2 min instead of 10+ sequential. Same for 2 agents studying scripts + researching BGE-M3.

2. **Pin Python ML dependencies aggressively** — FlagEmbedding + transformers version drift causes silent import failures. Always pin transformers version when installing ML packages.

3. **uv solves the "no sudo, no pip" problem** — Single curl install, no root needed, creates venvs without python3-venv package. Default tool for constrained environments.

4. **Pre-study the integration point** — Reading Dev's extracted JSON format + Supabase schema before writing the embedding script meant the code worked on first attempt. Schema-first pays off.
