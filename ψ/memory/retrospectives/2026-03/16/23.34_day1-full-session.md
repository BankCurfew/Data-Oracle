## Honest Feedback

**Friction 1: Cross-oracle handoff ยังไม่ smooth** — ผมต้อง manually source .env จาก Dev-Oracle ทุกครั้ง ต้อง maw hey ส่ง diagnosis ยาวๆ ที่อาจจะ truncate ไม่มี shared workspace หรือ PR-based collaboration ที่ทั้ง Data + Dev + QA เห็นพร้อมกัน Fix SQL อยู่ใน local file ของผม Dev ต้อง "เชื่อ" ว่ามันถูก

**Friction 2: kb_files cataloging ต้องทำ manual** — 518 files ต้อง categorize ทีละ batch ด้วย UPDATE statements ที่เขียนเอง ถ้ามี chunk_text → auto-categorize pipeline (Claude classify content → update metadata) จะเร็วกว่ามาก ตอนนี้ใช้ filename pattern matching ซึ่ง miss files ที่ชื่อไม่ตรง pattern

**Friction 3: ไม่มี way to test search quality systematically** — QA report ปัญหาเป็น case-by-case (ถาม PA ได้ Health Happy) แต่ไม่มี test suite ที่รัน "query X ต้องได้ result Y ไม่ได้ Z" ถ้ามี search quality regression test จะจับ bug ก่อน QA

## Lessons Learned

1. **Thai NLP stop words are critical** — คำเช่น "ประกัน", "คุ้มครอง" ต้อง exclude จาก ILIKE scoring ไม่งั้น match ทุก document
2. **bot_boost ที่สูงเกินไปเป็น anti-pattern** — x1.5 boost + common keywords = false positives ทุก query
3. **Category taxonomy enables filtering** — pa/health/ci/investment/savings/vitality — simple but effective
4. **Filename patterns cover ~80% of categorization** — ที่เหลือ 20% ต้อง content-based classification
5. **Cross-oracle collaboration needs shared artifacts** — SQL files, test cases, schema changes ควรอยู่ใน shared location
