## Honest Feedback

**Friction 1: Python environment chaos** — ไม่มี python, ไม่มี pip, ไม่มี pip3 ต้องลอง 4 commands ก่อนจะเจอ uv ที่ใช้ได้ ถ้ามี documentation ใน CLAUDE.md ว่า "ใช้ uv run python" จะประหยัดเวลาได้มาก ควรเพิ่มใน lessons learned สำหรับ oracle ตัวอื่นด้วย

**Friction 2: Supabase env location** — .env อยู่ที่ Dev-Oracle ไม่ใช่ Data-Oracle ทำให้ต้อง source จาก repo อื่น ไม่เป็น self-contained ถ้า Data-Oracle มี .env ของตัวเอง (หรือ symlink) จะสะดวกกว่า — แต่ต้องระวังเรื่อง secrets ใน git

**Friction 3: embed-chunks.py ไม่รองรับ admin-inserted data** — script อ่านจาก JSON files เท่านั้น ไม่ได้ออกแบบมาสำหรับ data ที่ INSERT ตรงเข้า Supabase ผมต้องเขียน embed-missing.py แยก ซึ่งก็ดีที่ได้เครื่องมือใหม่ แต่ถ้า original script มี mode "embed from db" จะครบวงจรกว่า

## Lessons Learned

1. **uv is the package manager** — ใช้ `uv run python` แทน python/python3 เมื่อต้องการ dependencies
2. **Parallel background tasks work great** — launch 5 sources parallel ลดเวลาจาก ~sequential 6000s เหลือ ~parallel 2900s
3. **Always audit after bulk operations** — หลัง embed เสร็จ ต้องเช็ค Storage URLs ด้วย ไม่ใช่แค่ embedding status
4. **Special characters in filenames = Storage upload failures** — `[`, `]`, `#`, `'`, Thai chars ทำให้ Supabase Storage upload ไม่ผ่าน
