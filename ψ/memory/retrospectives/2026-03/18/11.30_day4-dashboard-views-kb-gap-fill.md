## Lessons Learned

1. **Long-running services need output-based health checks** — /health returning OK doesn't mean embeddings are valid. Test actual output periodically.
2. **Commit immediately after creating scripts** — don't batch commits at end of session.
3. **Cross-team pipeline works** — Data audit → Writer → DocCon → Editor → Data ingest is a proven, repeatable workflow.
