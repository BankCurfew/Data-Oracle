## AI Diary

This was one of those sessions where everything connects. I started with a simple verification task — checking if aliases and placeholders were done — and ended up fixing a critical infrastructure bug that nobody knew existed.

The embedding service has been silently broken. Every embedding generated since PyTorch was updated to 2.10 has been NaN. That means the 35 chunks I found without valid embeddings during the KB audit weren't just "missing" — they were victims of this bug. The model weights were fine (567M params, zero NaN), but float32 arithmetic on CPU was overflowing at transformer layer 13 out of 24. The fix was simple — convert to float64 — but finding it required layer-by-layer tracing. I felt genuine satisfaction watching those embeddings come back as actual vectors instead of null arrays.

## Honest Feedback

**Friction 1: Supabase MCP permissions**. The MCP tool for Supabase SQL was completely blocked — had to fall back to REST API with curl. This added significant overhead to every query. The REST API requires counting with headers, can't do JOINs easily, and column discovery is trial-and-error. Need to either fix MCP permissions or build a proper query helper.

**Friction 2: Schema discovery**. I hit three different column name mismatches (content→chunk_text, file_name→filename, source_file doesn't exist, kb_documents→kb_files). There's no single schema reference. Every script has to discover columns by trial and error. A schema.md or auto-generated types would save enormous time.

**Friction 3: Embedding service silent failure**. The service was returning NaN vectors but reporting status "ok" on health check, latency numbers, and 200 status codes. There was zero indication anything was wrong. Need a validation step — embed a known test string on startup and verify the output has valid floats.

## Lessons Learned

1. **PyTorch 2.10 + CPU float32 = NaN at deep layers** — BGE-M3 (24 transformer layers) overflows on CPU with float32. Fix: `.double()` for float64. This is likely a regression in PyTorch's CPU attention implementation.
2. **Always validate embedding output** — health checks that don't verify output quality are useless. Embed a test string and check for NaN before marking service as "ready".
3. **REST API column discovery** — when Supabase MCP is blocked, query `?select=*&limit=1` to discover actual column names before building insert payloads.
